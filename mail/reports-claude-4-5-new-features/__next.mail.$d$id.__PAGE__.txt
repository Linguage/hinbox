<h2>Opus 4.5 相比 Opus 4.1 的主要改进</h2>
<h3>极致智力</h3>
<p>Claude Opus 4.5 代表了我们最智能的模型，结合了极致的能力与实用的性能。它在推理、编程和复杂问题解决任务上实现了阶梯式的提升，同时保持了 Opus 系列一贯的高质量输出。</p>
<h3>Effort 参数</h3>
<p>Claude Opus 4.5 是唯一支持 <a href="https://platform.claude.com/docs/en/build-with-claude/effort">Effort 参数</a>的模型，允许您控制 Claude 在响应时使用的 Token 数量。这使您能够通过单个模型在回复的详尽程度和 Token 效率之间进行权衡。</p>
<p>Effort 参数会影响响应中的所有 Token，包括文本回复、工具调用和扩展思维。您可以选择：</p>
<ul>
<li><strong>High effort（高）</strong>：最大程度的详尽性，适用于复杂的分析和详细的解释。</li>
<li><strong>Medium effort（中）</strong>：平衡的方法，适用于大多数生产用例。</li>
<li><strong>Low effort（低）</strong>：最节省 Token 的响应，适用于大批量自动化任务。</li>
</ul>
<h3>卓越的计算机使用能力</h3>
<p>Claude Opus 4.5 引入了<a href="https://platform.claude.com/docs/en/agents-and-tools/tool-use/computer-use-tool">增强的计算机使用能力</a>，新增了缩放（zoom）操作，能够以全分辨率详细检查特定的屏幕区域。这使得 Claude 能够检查在标准截图中可能不清楚的细微 UI 元素、小字体文本和详细的视觉信息。</p>

<ul>
<li>检查微小的 UI 元素和控件</li>

<li>在采取行动前验证精确的视觉细节</li>
</ul>

<h2>新 API 功能</h2>
<h3>编程式工具调用 (Beta)</h3>
<p><a href="https://platform.claude.com/docs/en/agents-and-tools/tool-use/programmatic-tool-calling">编程式工具调用</a>允许 Claude 在代码执行容器内通过编写代码以编程方式调用您的工具，而无需每次工具调用都经过模型的往返交互。这显著降低了多工具工作流的延迟，并通过允许 Claude 在数据到达模型上下文窗口之前对其进行过滤或处理来减少 Token 消耗。</p>
<pre><code class="language-python">tools=[
    {
        "type": "code_execution_20250825",
        "name": "code_execution"
    },
    {
        "name": "query_database",
        "description": "Execute a SQL query against the sales database. Returns a list of rows as JSON objects.",
        "input_schema": {...},
        "allowed_callers": ["code_execution_20250825"]  # Enable programmatic calling
    }
]
</code></pre>
<p>主要优势：</p>
<ul>
<li><strong>降低延迟</strong>：消除工具调用之间与模型的往返交互</li>
<li><strong>Token 效率</strong>：在返回给 Claude 之前，以编程方式处理和过滤工具结果</li>
<li><strong>复杂工作流</strong>：支持循环、条件逻辑和批处理</li>
</ul>
<p><a href="https://platform.claude.com/docs/en/agents-and-tools/tool-use/tool-search-tool">工具搜索工具</a>使 Claude 能够通过按需动态发现和加载工具来处理数百或数千个工具。Claude 不必预先将所有工具定义加载到上下文窗口中，而是搜索您的工具目录并仅加载其所需的工具。</p>
<p>提供两种搜索变体：</p>
<ul>
<li><strong>Regex</strong> (<code>tool_search_tool_regex_20251119</code>)：Claude 构建正则表达式模式来搜索工具名称、描述和参数</li>
<li><strong>BM25</strong> (<code>tool_search_tool_bm25_20251119</code>)：Claude 使用自然语言查询来搜索工具</li>
</ul>
<pre><code class="language-python">tools=[
    {
        "type": "tool_search_tool_regex_20251119",
        "name": "tool_search_tool_regex"
    },
    {
        "name": "get_weather",
        "description": "Get the weather at a specific location",
        "input_schema": {...},
        "defer_loading": True  # Load on-demand via search
    }
]
</code></pre>
<p>这种方法解决了两个关键挑战：</p>
<ul>
<li><strong>上下文效率</strong>：通过不预先加载所有工具定义，节省 10-20K Token</li>
<li><strong>工具选择准确性</strong>：即使有 100 多个可用工具，也能保持高准确性</li>
</ul>
<h3>Effort 参数 (Beta)</h3>
<p><a href="https://platform.claude.com/docs/en/build-with-claude/effort">Effort 参数</a>允许您控制 Claude 在响应时使用的 Token 数量，在响应详尽性和 Token 效率之间进行权衡：</p>
<pre><code class="language-python">response = client.beta.messages.create(
    model="claude-opus-4-5-20251101",
    betas=["effort-2025-11-24"],
    max_tokens=4096,
    messages=[{"role": "user", "content": "..."}],
    output_config={
        "effort": "medium"  # "low", "medium", or "high"
    }
)
</code></pre>
<p>Effort 参数会影响响应中的所有 Token，包括文本回复、工具调用和扩展思维。较低的 Effort 级别产生更简洁的响应和最少的解释，而较高的 Effort 则提供详细的推理和全面的答案。</p>
<h3>工具使用示例 (Beta)</h3>
<p><a href="https://platform.claude.com/docs/en/agents-and-tools/tool-use/implement-tool-use#providing-tool-use-examples">工具使用示例</a>允许您提供有效的工具输入具体示例，以帮助 Claude 了解如何更有效地使用您的工具。这对于具有嵌套对象、可选参数或格式敏感输入的复杂工具特别有用。</p>
<pre><code class="language-python">tools=[
    {
        "name": "get_weather",
        "description": "Get the current weather in a given location",
        "input_schema": {...},
        "input_examples": [
            {
                "location": "San Francisco, CA",
                "unit": "fahrenheit"
            },
            {
                "location": "Tokyo, Japan",
                "unit": "celsius"
            },
            {
                "location": "New York, NY"  # Demonstrates optional 'unit' parameter
            }
        ]
    }
]
</code></pre>
<p>示例包含在 Prompt 中与工具架构并列，向 Claude 展示格式良好的工具调用的具体模式。每个示例必须根据该工具的 <code>input_schema</code> 有效。</p>
<h3>记忆工具 (Beta)</h3>
<p>新的<a href="https://platform.claude.com/docs/en/agents-and-tools/tool-use/memory-tool">记忆工具</a>使 Claude 能够在上下文窗口之外存储和检索信息：</p>
<pre><code class="language-python">tools=[
    {
        "type": "memory_20250818",
        "name": "memory"
    }
]
</code></pre>
<p>这允许：</p>
<ul>
<li>随时间推移构建知识库</li>
<li>跨会话维护项目状态</li>
<li>通过基于文件的存储保留实际上无限的上下文</li>
</ul>
<h3>上下文编辑</h3>
<p>使用<a href="https://platform.claude.com/docs/en/build-with-claude/context-editing">上下文编辑</a>通过自动清除工具调用来进行智能上下文管理：</p>
<pre><code class="language-python">response = client.beta.messages.create(
    betas=["context-management-2025-06-27"],
    model="claude-sonnet-4-5",  # or claude-haiku-4-5
    max_tokens=4096,
    messages=[{"role": "user", "content": "..."}],
    context_management={
        "edits": [
            {
                "type": "clear_tool_uses_20250919",
                "trigger": {"type": "input_tokens", "value": 500},
                "keep": {"type": "tool_uses", "value": 2},
                "clear_at_least": {"type": "input_tokens", "value": 100}
            }
        ]
    },
    tools=[...]
)
</code></pre>
<p>此功能会在接近 Token 限制时自动删除较旧的工具调用和结果，帮助管理长期运行的智能体会话中的上下文。</p>
<h3>增强的停止原因</h3>
<p>Claude 4.5 模型引入了一个新的 <code>model_context_window_exceeded</code> 停止原因，明确指示生成何时因达到上下文窗口限制而停止，而不是请求的 <code>max_tokens</code> 限制。这使得在应用程序逻辑中更容易处理上下文窗口限制。</p>
<pre><code class="language-json">{
  "stop_reason": "model_context_window_exceeded",
  "usage": {
    "input_tokens": 150000,
    "output_tokens": 49950
  }
}
</code></pre>
<h3>改进的工具参数处理</h3>
<p>Claude 4.5 模型包含一个错误修复，可保留工具调用字符串参数中的预期格式。以前，字符串参数中的尾随换行符有时会被错误地剥离。此修复确保需要精确格式（如文本编辑器）的工具能够准确接收预期的参数。</p>
<p><strong>示例：</strong></p>
<pre><code class="language-json">// 之前：最后的换行符被意外剥离
{
  "type": "tool_use",
  "id": "toolu_01A09q90qw90lq917835lq9",
  "name": "edit_todo",
  "input": {
    "file": "todo.txt",
    "contents": "1. Chop onions.\n2. ???\n3. Profit"
  }
}

// 之后：尾随换行符按预期保留
{
  "type": "tool_use",
  "id": "toolu_01A09q90qw90lq917835lq9",
  "name": "edit_todo",
  "input": {
    "file": "todo.txt",
    "contents": "1. Chop onions.\n2. ???\n3. Profit\n"
  }
}
</code></pre>
<h3>Token 计数优化</h3>
<p>Claude 4.5 模型包含自动优化以提高模型性能。这些优化可能会向请求添加少量 Token，但<strong>您无需为这些系统添加的 Token 付费</strong>。</p>
<h2>Claude 4 引入的功能</h2>
<p>以下功能是在 Claude 4 中引入的，并适用于所有 Claude 4 模型，包括 Claude Sonnet 4.5 和 Claude Haiku 4.5。</p>
<h3>新的拒绝（Refusal）停止原因</h3>
<p>Claude 4 模型引入了一个新的 <code>refusal</code> 停止原因，用于模型出于安全原因拒绝生成内容的情况：</p>
<pre><code class="language-json">{
  "id": "msg_014XEDjypDjFzgKVWdFUXxZP",
  "type": "message",
  "role": "assistant",
  "model": "claude-sonnet-4-5",
  "content": [{"type": "text", "text": "I would be happy to assist you. You can "}],
  "stop_reason": "refusal",
  "stop_sequence": null,
  "usage": {
    "input_tokens": 564,
    "cache_creation_input_tokens": 0,
    "cache_read_input_tokens": 0,
    "output_tokens": 22
  }
}
</code></pre>
<p>使用 Claude 4 模型时，您应该更新应用程序以<a href="https://platform.claude.com/docs/en/test-and-evaluate/strengthen-guardrails/handle-streaming-refusals">处理 <code>refusal</code> 停止原因</a>。</p>
<h3>摘要式思维 (Summarized thinking)</h3>
<p>启用扩展思维后，Claude 4 模型的 Messages API 会返回 Claude 完整思维过程的摘要。摘要式思维提供了扩展思维的全部智力优势，同时防止滥用。</p>
<p>虽然 API 在 Claude 3.7 和 4 模型之间是一致的，但扩展思维的流式响应可能会以“块状”模式传输，流式事件之间可能存在延迟。</p>
<p>有关更多信息，请参阅<a href="https://platform.claude.com/docs/en/build-with-claude/extended-thinking#summarized-thinking">扩展思维文档</a>。</p>
<h3>穿插思维 (Interleaved thinking)</h3>
<p>Claude 4 模型支持将工具使用与扩展思维穿插进行，允许进行更自然的对话，其中工具使用和回复可以与常规消息混合。</p>
<p>有关更多信息，请参阅<a href="https://platform.claude.com/docs/en/build-with-claude/extended-thinking#interleaved-thinking">扩展思维文档</a>。</p>
<h3>行为差异</h3>
<p>Claude 4 模型有明显的行为变化，可能会影响您构建 Prompt 的方式：</p>
<h4>沟通风格的变化</h4>
<ul>
<li><strong>更简洁直接</strong>：Claude 4 模型的沟通效率更高，解释更少冗长</li>
<li><strong>更自然的语气</strong>：回复稍微更具对话性，减少了机器味</li>
<li><strong>注重效率</strong>：完成动作后可能会跳过详细总结以保持工作流势头（如果需要，您可以通过 Prompt 要求更多细节）</li>
</ul>
<h4>指令遵循</h4>
<p>Claude 4 模型经过训练可进行精确的指令遵循，需要更明确的指导：</p>
<ul>
<li><strong>明确行动</strong>：如果您希望 Claude 采取行动，请使用直接的语言，如“进行这些更改”或“实施此功能”，而不是“您可以建议更改吗”</li>
<li><strong>清晰说明期望行为</strong>：Claude 将精确遵循指令，因此具体说明您想要什么有助于获得更好的结果</li>
</ul>
<p>有关使用这些模型的全面指导，请参阅 <a href="https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-4-best-practices">Claude 4 Prompt 工程最佳实践</a>。</p>
<h3>更新的文本编辑器工具</h3>
<p>文本编辑器工具已针对 Claude 4 模型进行了更新，更改如下：</p>
<ul>
<li><strong>工具类型</strong>：<code>text_editor_20250728</code></li>
<li><strong>工具名称</strong>：<code>str_replace_based_edit_tool</code></li>
<li>不再支持 <code>undo_edit</code> 命令</li>
</ul>
<p>如果您从 Claude Sonnet 3.7 迁移并正在使用文本编辑器工具：</p>
<pre><code class="language-python"># Claude Sonnet 3.7
tools=[
    {
        "type": "text_editor_20250124",
        "name": "str_replace_editor"
    }
]

# Claude 4 Models
tools=[
    {
        "type": "text_editor_20250728",
        "name": "str_replace_based_edit_tool"
    }
]
</code></pre>
<p>有关更多信息，请参阅<a href="https://platform.claude.com/docs/en/agents-and-tools/tool-use/text-editor-tool">文本编辑器工具文档</a>。</p>
<h3>更新的代码执行工具</h3>
<p>如果您正在使用代码执行工具，请确保使用最新版本 <code>code_execution_20250825</code>，该版本添加了 Bash 命令和文件操作功能。</p>
<p>旧版本 <code>code_execution_20250522</code>（仅 Python）仍然可用，但不建议用于新实现。</p>
<p>有关迁移说明，请参阅<a href="https://platform.claude.com/docs/en/agents-and-tools/tool-use/code-execution-tool#upgrade-to-latest-tool-version">代码执行工具文档</a>。</p>
